加密性能对比

对比 AES 与 Paillier 在不同数据规模下的接口返回指标，重点观察“速度、吞吐、隐私能力”的取舍，作为后续自适应优化的参数成本参考。

AES-256 对称加密
适合批量数据传输和存储加密，速度快、吞吐高，但不支持在密文上直接计算。系统中用于说明传统加密的效率优势。
Paillier 同态加密
支持加法同态，可在不解密的情况下做部分统计或聚合，隐私保护能力更强，但加解密耗时和计算开销明显更高。

实验报告 - 项目技术路线与答辩总结
项目概述

本系统是面向密码攻击检测场景的大创原型系统，重点展示公开数据集接入、联邦节点划分、攻击检测、参数优化和安全可观测链路。

技术路线

公开数据集 → 特征提取 → 四节点划分 → 联邦训练 → 攻击检测 → 参数优化 → 安全事件追踪。

当前边界

当前为 Flask + SQLite 的原型系统；联邦训练在本地运行四节点流程；安全防护模块展示已接入的请求追踪、慢接口、限流和事件查询能力。

模块能力表
模块当前能力展示方式

联邦训练本地训练基线与四节点聚合流程查看节点样本数、执行一轮联邦训练
攻击检测从已处理数据或上传文件调用检测接口展示风险分数、风险等级和攻击类型
