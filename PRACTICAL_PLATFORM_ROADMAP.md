# 密码攻击检测与隐私训练平台实用化改造路线

## 1. 项目定位

项目对外标题固定为：**密码攻击检测与隐私训练平台**。

系统定位为面向账号登录安全、密码攻击风险识别和隐私保护训练的轻量双端平台：

- 用户端：上传数据 -> 风险检测 -> 分析报告。
- 管理端：用户提交 -> 数据处理 -> 模型版本 -> 系统审计。

训练能力服务于检测模型更新，不单独作为产品主线。联邦训练用于展示高隐私数据不能集中时的协同建模流程，Paillier 用于安全聚合和密态字段展示，AES 用于上传文件加密归档。

页面文案不使用“实验、演示、模拟演示、答辩、原型展示”。可以使用“当前能力、当前数据源、训练数据池、四节点流程、本地训练、联邦训练、安全聚合、当前检测模型、模型版本、平台边界”等表达。

## 2. 参考项目取舍

参考项目：[Network-Security-Based-On-ML](https://github.com/zimingttkx/Network-Security-Based-On-ML)

### 2.1 可参考思想

- 统一检测管道：不同检测来源统一进入风险输出结构。
- 多检测来源融合：规则、行为字段、轻量模型结果共同形成风险分数。
- 检测结果统一字段：风险等级、攻击类型、置信度、触发特征、建议动作统一输出。
- 检测统计看板：展示风险等级分布、攻击类型分布、检测次数和模型状态。
- 模型状态展示：展示当前运行模型、训练来源、指标、模型文件状态。
- 数据集管理：将不同数据源转换为统一字段后再训练。
- 检测结果排序与风险分布：用风险排名替代固定 Top 样本。

### 2.2 不能照搬内容

- TensorFlow / PyTorch 深度学习栈。
- Kitsune / LUCID 完整实现。
- DQN / PPO 强化学习模块。
- Kubernetes、Redis、Celery、MLflow 等重型平台组件。
- WAF、防火墙、扫描器、漏洞扫描和端口封禁逻辑。
- 大而全网络安全平台叙事。

本项目只吸收“管道化、统一结果、轻量看板、模型状态、数据集导入”这些设计思想，并落地到 Flask + SQLite + 原生前端。

## 3. 用户端结构

用户端只保留三页：

1. 上传数据
2. 风险检测
3. 分析报告

### 3.1 上传数据

目标：让用户提交账号登录安全数据，并看到加密归档和脱敏预览。

页面保留：

- 上传 CSV / JSON。
- 生成登录安全样本。
- 生成隐私加密样本。
- 提交编号、样本数、字段数、缺失率。
- 登录安全字段识别。
- AES 加密归档状态。
- 样本脱敏预览。
- Paillier 密态字段展示。

约束：

- username / phone / email / token / password / salary / id_card 等不能明文展示。
- password / token / secret 永不展示明文，只展示派生特征。
- 隐私字段不直接作为攻击检测原因。

### 3.2 风险检测

目标：展示统一检测管道输出，不再规划独立“风险详情”模块。

删除：

- 风险详情模块。
- 展开详情按钮。
- 单条详情面板。
- Top 20 文案。

页面改为：

- 风险概览：总样本、高风险、中风险、低风险、平均风险分、当前检测模型。
- 检测统计图：风险等级分布、攻击类型分布、风险分数区间分布、主要触发因素统计。
- 风险排名表：按 risk_score 从高到低排序，默认展示前 100 条。

风险排名字段：

| 字段 | 说明 |
| --- | --- |
| 排名 | 按 risk_score 降序 |
| 样本编号 | 上传数据或系统测试集样本编号 |
| 用户 | 脱敏用户标识 |
| IP | 来源 IP |
| 风险等级 | low / medium / high / critical |
| 风险分数 | 0 到 1 |
| 攻击类型 | 疑似暴力破解、高频请求异常等 |
| 主要因素 | failed_attempts、request_frequency 等 |
| 建议 | 建议动作 |

### 3.3 分析报告

目标：报告跟随风险排名，不再出现 Top 20。

报告内容：

- 提交编号。
- 文件名。
- 总样本。
- 高 / 中 / 低风险数量。
- 主要风险类型。
- 当前检测模型。
- 加密归档状态。
- 敏感字段。
- 风险排名摘要。
- 处理建议。

要求：

- Markdown 可下载。
- 不泄露明文敏感字段。
- 不出现“实验 / 演示 / 模拟演示 / 答辩 / 原型展示”。

## 4. 管理端结构

管理端导航改为：

1. 用户提交
2. 数据处理
3. 模型版本
4. 系统审计
5. 退出

### 4.1 用户提交

定位：只展示用户上传数据、风险摘要、审核状态和训练准入。

内容：

- 用户提交数量。
- 高风险样本。
- 中风险样本。
- 可训练提交。
- 用户提交表格：时间、文件、样本 / 字段、安全字段、加密、审核状态、训练状态、最近操作、风险摘要、操作。

操作：

- 归档。
- 拒绝。
- 标记可训练。

用户提交模块不展示系统训练数据源。

### 4.2 数据处理

定位：训练数据源、公开数据集状态、四节点准备状态和训练入口。

内容：

- 当前训练数据摘要。
- 系统训练数据源表格。
- 公开数据集导入状态。
- 四节点准备状态。
- 本地训练与四节点联邦训练入口。

四节点字段：

- hospital：医疗节点。
- bank：金融节点。
- insurance：保险节点。
- government：政务节点。

每个节点展示样本数、正常 / 攻击数量、特征数、来源、ready / missing。

### 4.3 模型版本

定位：展示当前检测模型、训练任务记录和模型版本状态。

当前检测模型状态：

- 当前检测模型。
- 模型类型。
- 算法。
- 数据来源。
- 是否可用。
- 指标。
- 最后更新时间。
- 模型文件状态。

模型版本列表：

- 模型版本。
- 来源。
- 指标。
- 创建时间。
- 模型文件状态。
- 当前启用状态。
- 操作。

规则：

- 有运行时模型文件：允许设为检测模型。
- 只有训练记录：只作为训练追踪版本。
- 文件缺失：禁用并显示原因。
- 不做假回退。

### 4.4 系统审计

定位：只记录平台访问和平台运行安全事件，不混入用户上传数据风险。

内容：

- 安全事件：慢接口、限流、中高风险访问、异常路径。
- 最近访问记录：IP、归属地、设备、系统、浏览器、路径、TraceId。

要求：

- 安全事件和访问流水分开统计。
- 低 / 中 / 高风险统计与事件列表一致。
- 详情点击有效。

## 5. 轻量数据集导入方案

### 5.1 支持数据源

- 当前 `data/generated/train.csv`
- 当前 `data/generated/test.csv`
- UNSW-NB15
- CIC-IDS2017
- CSE-CIC-IDS2018
- CIC-DDoS2019

### 5.2 小服务器约束

- CSV 分块读取。
- 每类样本默认最多 20000 条。
- 不一次性加载全量大文件。
- 不训练深度学习模型。
- 不提交原始数据、处理后 CSV、模型文件、日志、数据库。

### 5.3 统一字段

| 字段 | 说明 |
| --- | --- |
| sample_id | 样本编号 |
| source_dataset | 数据集来源 |
| attack_type | 攻击类型 |
| label | 0 正常 / 1 风险 |
| src_ip | 源 IP |
| dst_ip | 目标 IP |
| protocol | 协议 |
| flow_duration | 流持续时间 |
| total_fwd_packets | 正向包数 |
| total_bwd_packets | 反向包数 |
| flow_bytes_s | 每秒字节数 |
| flow_packets_s | 每秒包数 |
| request_frequency | 请求频率 |
| response_time | 响应时间 |
| failed_attempts | 失败次数 |
| unusual_hour | 异常时间 |
| payload_size | 负载大小 |
| device_type | 设备类型 |
| browser | 浏览器 |
| os | 操作系统 |
| username_masked | 脱敏用户 |

缺失字段处理：

- 数值字段填 0。
- 文本字段填 `-`。
- 不因单个字段缺失中断导入。

### 5.4 标签映射

- Benign / normal -> label 0。
- Brute Force / FTP-Patator / SSH-Patator / SSH-Bruteforce / FTP-BruteForce / Web Attack-Brute Force -> password_attack。
- DDoS / DoS / PortScan / Botnet / Infiltration / Exploit 等 -> anomaly_attack。
- 其他攻击类型保留原始 attack_type，但 label 设为 1。

## 6. 统一检测管道方案

### 6.1 检测来源

- 规则评分。
- 登录失败次数评分。
- 请求频率评分。
- 异常时间评分。
- 响应时间评分。
- 设备 / IP 异常评分。
- 当前轻量模型评分。

### 6.2 输出字段

```json
{
  "is_risk": true,
  "risk_score": 0.82,
  "risk_level": "high",
  "attack_type": "疑似暴力破解",
  "confidence": 0.91,
  "action_suggestion": "强制改密并开启二次验证",
  "detection_time_ms": 12.3,
  "trigger_features": ["failed_attempts", "request_frequency"],
  "score_breakdown": {
    "failed_attempts_score": 0.25,
    "request_frequency_score": 0.20,
    "unusual_time_score": 0.10,
    "response_time_score": 0.05,
    "device_ip_score": 0.12,
    "model_score": 0.10
  },
  "reason": "登录失败次数和请求频率偏高。",
  "suggestion": "建议提醒用户改密并开启二次验证。",
  "source_dataset": "user_submission",
  "model_version": "v20260622"
}
```

### 6.3 风险等级和动作建议

| 风险等级 | 建议动作 |
| --- | --- |
| low | 观察 |
| medium | 提醒用户改密 |
| high | 强制改密并开启二次验证 |
| critical | 临时冻结账号并人工复核 |

目标：

- 每条检测结果包含 risk_score、risk_level、attack_type、confidence、trigger_features、score_breakdown、reason、suggestion、source_dataset、model_version。

### Task 4：用户端风险检测页和报告页

目标：

- 删除风险详情模块。
- 风险检测页内部展示风险排名。
- 报告页展示风险排名摘要。

- 用户端只有上传数据、风险检测、分析报告三页。
- 没有 Top 20 文案。

### Task 5：管理端导航重构

目标：

- 管理端导航改为用户提交 / 数据处理 / 模型版本 / 系统审计 / 退出。

验收：

- 用户提交只展示上传数据。
- 数据处理承接数据源、导入状态、四节点、训练入口。
- 模型版本独立展示当前检测模型和训练记录。
- 系统审计保持独立。

### Task 6：数据处理模块接入公开数据集状态

目标：

- 管理端能看到公开数据集是否已下载、是否已处理、是否可训练。

验收：

- UNSW-NB15 / CIC-IDS2017 / CSE-CIC-IDS2018 / CIC-DDoS2019 状态清晰。
- 未配置时显示未配置。
- 已处理时显示样本数、字段数、标签列和攻击类型分布。

### Task 7：模型版本和当前检测模型状态优化

目标：

- 模型版本功能可解释，不做假回退。

验收：

- 当前检测模型状态不空白。
- 训练追踪版本和可启用检测模型区分明确。
- 没有运行时模型文件时禁用“设为检测模型”并说明原因。
- 可启用版本能真实影响后续检测模型选择。

## 9. 最终目标

用户端：

上传数据 -> 风险检测 -> 风险排名 -> 分析报告。

管理端：

用户提交 -> 数据处理 -> 模型版本 -> 系统审计。

数据能力：

支持本地生成数据 + UNSW-NB15 + CIC-IDS2017 + CSE-CIC-IDS2018 + CIC-DDoS2019 的轻量导入和统一字段处理。

检测能力：

采用统一风险结果格式，多来源分数融合，展示分数构成、触发字段、风险原因和建议动作。

平台风格：

简洁、实用、稳定、可部署，不堆概念，不出现“实验 / 演示 / 答辩”这类词。
