# 密码攻击检测与隐私训练平台开发文档

## Summary

项目对外标题固定为：**密码攻击检测与隐私训练平台**。

推荐定位：

> 一个面向账号登录安全、密码攻击风险识别和隐私保护训练的双端平台。

产品主线：

```text
用户端：
上传账号安全数据
→ 密码攻击风险检测
→ 风险原因解释
→ 处置建议
→ 分析报告

管理端：
加密归档数据
→ 数据审核
→ 本地模型训练
→ 四节点联邦训练
→ 模型版本管理
→ 系统审计
```

核心技术主线：

```text
AES 加密归档
+ Paillier 密态保护 / 安全聚合展示
+ FedAvg 四节点联邦训练
+ 密码攻击风险检测
+ 模型版本追踪
+ 系统审计可观测
```

文案策略：

- 可以强调“平台化”“隐私训练”“密码攻击风险检测”“多模型融合”“联邦训练能力”“Paillier 安全聚合能力”。
- 不写成“检测所有网络攻击”“完整生产级 WAF”“完整密文机器学习”“真实跨机构生产联邦学习”。
- DDoS 只能作为扩展方向或高频访问异常识别能力，不作为当前核心能力。

```text
密码攻击检测与隐私训练平台
```

```text
本系统面向高隐私账号安全场景，提供用户数据上传、密码攻击风险检测、原因解释、报告生成、加密归档、联邦训练和系统审计能力。
```

允许使用的增强表达：

- 密码攻击风险识别
- 账号异常检测

## Task R2：用户端强化为“密码攻击风险检测流程”

目标：用户端像真实使用入口，而不是实验页面。

用户端保留三步：

```text
上传数据
风险检测
分析报告
```

上传数据页展示：

- 上传 CSV / JSON。
- 生成登录安全样本。
- 生成隐私加密样本。
- AES 加密归档状态。
- 敏感字段识别结果。
- 脱敏样本预览。
- 样本预览必须包含账号安全字段，例如用户名、IP、设备、浏览器、系统、失败次数、请求频率、响应时间、时间戳、标签。

风险检测页展示：

- 总样本数。
- 高风险 / 中风险 / 低风险数量。
- 密码攻击风险分布。
- 攻击类型分布。
- 高风险样本 Top 20。
- 风险原因。
- 处置建议。
- 当前使用模型版本。

分析报告页展示：

- 数据概况。
- 风险摘要。
- 高风险样本。
- 密码攻击风险原因。
- 处置建议。
- 隐私保护说明。
- 报告下载。

推荐文案：

```text
系统根据登录失败次数、请求频率、响应耗时、访问设备、IP 来源和标签字段，识别可能存在的暴力破解、异常登录、脚本化访问和账号风险行为。
```

验收标准：

- 用户可以完整走通：上传 → 检测 → 报告。
- 风险原因必须来自登录安全字段。
- 手机号、收入、信用分等隐私字段不直接作为攻击检测原因。

## Task R3：攻击检测能力包装为“密码攻击风险检测”

目标：避免“DDoS 检测不真实”的问题，把主能力聚焦到密码攻击和账号风险。

当前检测主线定义为：

```text
密码攻击风险检测 / 账号登录异常检测
```

可识别风险类型：

- 暴力破解风险。
- 高频登录请求。
- 异常失败次数。
- 异常响应耗时。
- 异常设备或浏览器。
- 异常 IP 来源。
- 登录行为与标签不一致。
- 可疑脚本化访问。

攻击类型展示建议：

```text
正常访问
疑似暴力破解
高频请求异常
异常设备登录
账号风险行为
可疑自动化访问
```

DDoS 表述方式：

```text
系统当前可通过请求频率、响应耗时和访问来源特征识别部分高频访问异常；更完整的网络层 DDoS 检测需要接入流量级数据集和时间窗口聚合模块。
```

验收标准：

- 页面不再把系统说成完整 DDoS 平台。
- 可以保留“高频访问异常 / 可疑自动化访问”作为展示能力。
- 风险检测结果更像密码攻击检测系统。

## Task R5：添加攻击检测相关数据集

目标：让攻击检测和联邦训练有清晰数据来源，不再只依赖不明来源样本。

数据源分三类：

### 1. 内置密码攻击训练集

使用当前项目已有数据：

```text
data/generated/train.csv
data/generated/test.csv
```

定位：

```text
系统内置密码攻击训练样本库
```

用途：

- 用户端默认检测样本来源。
- 管理端初始训练数据源。
- 本地训练。
- 四节点联邦训练。
- 模型版本生成。

### 2. 公开入侵检测数据集

接入 UNSW-NB15。

用途：

- 扩展异常流量、扫描、入侵检测类风险识别。
- 作为管理端可选训练数据源。

要求：

- 如果服务器未配置 UNSW-NB15，页面显示“未配置公开数据集”。
- 不伪装成已经加载。
- 不把 UNSW-NB15 写成项目自有真实业务数据。

### 3. DDoS / 流量型攻击数据集

后续接入 CIC-IDS2017 或 CIC-DDoS2019。

要求：

- 必须先实现时间窗口特征提取。
- 当前系统不直接宣称完整 DDoS 检测能力。
- 当前只能说支持高频访问异常识别和后续 DDoS 数据集扩展。

管理端数据源页新增：

- 数据源名称。
- 数据源类型。
- 样本总数。
- 字段数。
- 标签列。
- 攻击类型分布。
- 是否可训练。
- 是否已生成联邦节点。

验收标准：

- 管理端能清楚看到当前训练数据来自哪里。
- 攻击检测能选择系统内置测试集或用户上传数据。
- 不再出现“没有数据集但页面还显示已训练”的问题。

## Task R6：解释四节点联邦训练中“500 条数据”的真实含义

目标：解决“训练中心四个联邦 500 条到底是什么，有没有用”的问题。

当前解释：

```text
四个节点的 500 条数据不是四个真实机构的数据，而是系统训练数据池经过标签分层切分后得到的四个联邦节点数据。
```

页面说明建议：

```text
当前四节点数据由训练数据池按标签分层划分得到，用于模拟高隐私数据不能集中时的联邦训练流程。每个节点显示的样本数来自实际切分结果，不代表真实医院、银行、保险、政府机构的数据规模。
```

四节点卡片展示：

- 节点名称：hospital / bank / insurance / government。
- 节点类型：模拟联邦节点。
- 样本数。
- 正常样本数。
- 攻击样本数。
- 攻击类型分布。
- 特征维度。
- 数据来源。
- 是否参与本轮训练。

作用说明：

- 有用：验证数据划分、本地训练、FedAvg 聚合、模型版本生成链路。
- 不够：不能证明真实跨机构部署，也不能证明真实机构数据隔离训练。
- 下一步：如果要更真实，需要多进程或多服务节点部署，每个节点独立持有数据。

验收标准：

- 用户能看懂四节点 500 条从哪里来。
- 样本数不是前端写死，而是来自后端实际节点数据。
- 训练中心明确标注“模拟联邦节点”。

## Task R7：Paillier 安全聚合展示增强

目标：让 Paillier 成为项目亮点，而不是只在加密对比里出现。

训练中心新增“聚合方式”选择：

```text
普通 FedAvg
Paillier 安全聚合
```

Paillier 模式展示：

- 参数量化。
- Paillier 加密。
- 密态聚合。
- 聚合结果解密。
- 加密耗时。
- 解密耗时。
- 聚合耗时。
- 精度差异。
- 安全说明。

推荐展示链路：

```text
节点本地训练
→ 参数/梯度量化
→ Paillier 加密
→ 服务端密态加权聚合
→ 解密全局参数
→ 生成模型版本
```

联邦训练接口建议扩展：

```json
{
  "epochs": 3,
  "limit": 10000,
  "aggregation_method": "paillier",
  "secure_aggregation": true
}
```

返回字段建议：

```json
{
  "paillier_enabled": true,
  "encrypted_parameter_count": 120,
  "encryption_time_ms": 83.2,
  "aggregation_time_ms": 41.5,
  "decryption_time_ms": 35.8,
  "accuracy_delta": 0.012
}
```

验收标准：

- 页面能直观看到 Paillier 聚合流程。
- 能体现“安全增强但性能开销更高”。
- Paillier 初始化失败时有明确提示。

## Task R10：报告页增强为“密码攻击风险分析报告”

报告结构：

```text
1. 报告摘要
```

## Task R11：系统审计区分“平台安全”和“用户数据风险”

目标：避免审计和攻击检测混淆。

系统审计说明：

```text
系统审计记录谁访问了平台，以及平台运行中出现的安全事件。
用户风险检测分析的是用户上传的数据。
两者相互独立。
```

安全事件展示：

- 高风险访问。
- 中风险访问。
- 慢接口。
- 限流。
- 异常路径。
- 可疑 IP。
- 中文原因。
- 处理建议。

最近访问记录展示：

- IP。
- 归属地。
- 设备。
- 系统。
- 浏览器。
- 请求路径。
- TraceId。

验收标准：

- 安全事件不混入普通访问记录统计。
- 最近访问记录只作为访问流水。
- 详情点击有效。
- 低风险、中风险、高风险统计与事件列表一致。
